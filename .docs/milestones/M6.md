# M6 — AVP Splat Editor: deferred polish + Metal/BE-heavy work

Picks up the 11 tasks deferred from M5. Three shape clusters:

1. **Metal geometry / shader work** — gizmos, previews, perf HUDs.
2. **MetalSplatter-fork-dependent** — per-splat mask attribute work that blocks erase-restore + stamp tooling.
3. **New surface area** — hotspot tool, floor-plan in AVP, splat upload, multi-editor presence, comments.

Each cluster ships independently; no implicit M6.x → M6.y dependencies inside a cluster except where listed.

## Priority bands

- **P0** — feature parity with web (hotspot tool, splat upload).
- **P1** — high leverage editor polish (Metal geometry, snap, perf).
- **P2** — collaboration / large-feature scope (presence, comments, floor-plan).

## Tasks

### P0 — Feature parity with web

- [x] **M6.1** — Hotspot tool (was M5.4)
  - **Goal:** New `ToolMode.hotspot`. Aim + pinch places a hotspot; second pinch cycles content type (link / image / text). SwiftUI side-panel edits the chosen content. Save persists to `hotspots` table via existing BE route.
  - **Files:** `EditTool.swift` (case `.hotspot`), `SplatImmersiveRenderer.swift` (gesture + render), `SplatViewerView.swift` (picker + content panel), new `cloudtour-avp/CloudTourVision/Models/Hotspot.swift`. BE wiring already exists at `cloudtour-be/src/routes/scenes/[sceneId]/hotspots`.
  - **Depends on:** M5.2 (shipped).

- [x] **M6.2** — Splat upload from device (was M5.17)
  - **Goal:** AVP can pick a `.ply` / `.splat` / `.spz` from Files, validate magic bytes, upload to Supabase storage, and create a new scene attached to a tour. Currently web-only.
  - **Files:** new `cloudtour-avp/CloudTourVision/Dashboard/SplatUploadView.swift`, BE upload route reuse (`cloudtour-be/src/routes/org-tours.ts`).

### P1 — Calibrate ergonomics

- [x] **M6.3** — Snap-to-floor + snap-to-grid (was M5.7)
  - **Goal:** Toggle in calibrate panel. Floor snap aligns lowest splat-AABB face to y=0; grid snap quantises translation to 5 cm.
  - **Files:** `SplatImmersiveRenderer.swift` (snap logic in `applyTwoHandGesture` + `applyOneHandGesture`).
  - **Note:** AABB computed at `load()` time from `[SplatScenePoint]` rather than via the upstream fork — no M4.16 dependency.

- [x] **M6.4** — Axis gizmo overlay (was M5.8)
  - **Goal:** While `.calibrate` is active, render a small XYZ gizmo at the splat origin so the user can see orientation drift without external cues.
  - **Files:** `SplatImmersiveRenderer.swift` (gizmo geometry alongside silhouette), shader registration.

### P1 — Waypoints

- [x] **M6.5** — Arrival-yaw preview arrow (was M5.9)
  - **Goal:** Before committing a yaw-update pinch, show a translucent arrow at the waypoint indicating the candidate yaw. Pinch confirms.
  - **Files:** `SplatImmersiveRenderer.swift`.

### P1 — Display / debug

- [x] **M6.6** — Performance counters (was M5.12)
  - **Goal:** fps + draw-count overlay, toggled by HUD or shake gesture. Off by default.
  - **Files:** `SplatImmersiveRenderer.swift` (frame timing via `LayerRenderer.Drawable.frameTiming`), HUD wiring already in `SplatViewerView.swift`.

- [ ] **M6.7** — Display HUD: opacity + point-size sliders (was M5.11 partial) — **deferred to M7 (blocked on M4.16 fork execute)**
  - **Goal:** Continuous splat opacity (0.1–1.0) and point-size multiplier (0.5–2.0) sliders in the existing display HUD. Renderer reads as shader uniforms.
  - **Files:** `SplatImmersiveRenderer.swift` (uniform plumbing through MetalSplatter render path), `SplatViewerView.swift` HUD.
  - **Depends on:** likely M4.16 if MetalSplatter doesn't expose those uniforms upstream.

### P1 — Selection refinement

- [ ] **M6.8** — Erase-the-eraser (was M5.13) — **deferred to M7 (blocked on M4.16 fork execute)**
  - **Goal:** Modifier (long pinch) inside a pending deletion volume restores splats in that sub-region. Implemented as a positive-mask deletion that subtracts from the negative.
  - **Files:** `SplatImmersiveRenderer.swift` (DeletionMask diff), `EditTool.swift`.
  - **Depends on:** M4.16 (per-splat mask attribute).

- [x] **M6.9** — Lasso projection-cone preview (was M5.14)
  - **Goal:** While drawing the lasso, render a translucent extrusion of the polygon along the view direction so the user sees what will be erased before release.
  - **Files:** `SplatImmersiveRenderer.swift`.

### P1 — A11y / sim parity completion

- [x] **M6.10** — VoiceOver full audit (was M5.21 partial)
  - **Goal:** Every interactive control has an accessibility label + hint. Custom Metal overlays expose accessibility hit-rects where they replace native UI. Tested with VoiceOver on visionOS device.
  - **Files:** `SplatViewerView.swift`, `SplatImmersiveRenderer.swift` (a11y proxy nodes if needed).

- [x] **M6.11** — Sim parity: box / lasso commit shortcuts (was M5.22 partial)
  - **Goal:** Keyboard shortcuts in the simulator that synthesize the gesture-state needed to commit a box (dual-pinch release) and a lasso (single-pinch release after sample).
  - **Files:** `SplatImmersiveRenderer.swift` (`debugTriggerBoxCommit`, `debugTriggerLassoCommit`), `SplatViewerView.swift` sim controls.

### P1 — Waypoint graph 2D layout

- [x] **M6.12** — Scene-link graph 2D layout (was M5.10 deferred portion)
  - **Goal:** True force-directed 2D layout of scenes-as-nodes / waypoints-as-edges. Replaces the M5.10 list-style fallback when the scene count exceeds ~12.
  - **Files:** `cloudtour-avp/CloudTourVision/Dashboard/WaypointGraphView.swift`.

### P2 — Collaboration / large-feature scope — **all deferred to M7**

Deliberate defer (per acceptance criteria — "P2 explicitly chosen-or-deferred"). Reason: collaboration cluster (Realtime presence, comments table + BE routes, in-immersive metadata edit, floor-plan editor) is roughly half a milestone of its own and would dilute the M6 polish focus. M7 takes the full P2 cluster as its primary scope.

- [ ] **M6.13** — Floor-plan editor in AVP (was M5.15)
  - **Goal:** AVP UI to drag scenes onto a tour-level 2D floor-plan, persisting normalized `[{scene_id, x, y}]` to `tours.scene_positions`.
  - **Files:** new `cloudtour-avp/CloudTourVision/Dashboard/FloorPlanEditorView.swift`. BE already exists.

- [ ] **M6.14** — Tour metadata quick-edit in-immersive (was M5.16)
  - **Goal:** Side-panel inside immersive view to edit title / category / status without exiting. Reuses existing BE PATCH.
  - **Files:** `SplatViewerView.swift`, `cloudtour-avp/CloudTourVision/ViewModels/TourDetailViewModel.swift`.

- [ ] **M6.15** — Multi-editor presence + conflict detection (was M5.18)
  - **Goal:** Show other editors' cursors/avatars in the same scene; warn on conflict at Save (server returns 409 with diff).
  - **Files:** Supabase Realtime channel per `(tour, scene)`, new `cloudtour-avp/CloudTourVision/Realtime/EditorPresence.swift`. BE: optimistic-lock check on `scene_edits.version`.

- [ ] **M6.16** — Comments / annotation overlay (was M5.19)
  - **Goal:** Pinch-anchored comment threads in splat-local space. Visible in viewer mode (read-only) and editor mode (read-write).
  - **Files:** new `comments` table + RLS, BE routes, AVP comment marker + thread UI.

- [ ] **M6.17** — Stamp / clone tool (was M5.23) — **deferred to M7 (blocked on M4.16 fork execute)**
  - **Goal:** Select region → save as brush → stamp elsewhere with blend cap + jitter (scale + in-plane rot).
  - **Depends on:** M4.16 (fork w/ mutable splat buffers), M5.1 (shipped).
  - **Reference:** `.docs/architecture/splatpainting-evaluation.md` — full rationale for not integrating upstream; rebuild from concept only.
