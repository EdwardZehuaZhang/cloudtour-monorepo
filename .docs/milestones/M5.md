# M5 — AVP Splat Editor: polish + parity

Closes the gaps surfaced in the M4.15 editor audit. Goal was "feels like a real editor" — undo, per-item control, hotspots, display toggles, autosave — without re-architecting the renderer.

11 of 22 originally-scoped tasks shipped this session. The remaining 11 (Metal-heavy, fork-dependent, or large-feature scope) are tracked in `.docs/milestones/M6.md` as M6.1–M6.17 with full spec retained.

## Shipped

### P0 — Save-flow trust

- [x] **M5.1** — Undo / redo stack — _2026-04-27_
  - HistorySnapshot deep-copy on every commit (waypoint pinch, brush sphere, box release, lasso release, applyTransform). Undo/redo public API + Cmd-Z / Cmd-Shift-Z. Stack capped at 64 entries.
  - **Files:** `SplatImmersiveRenderer.swift`, `SplatViewerView.swift`.

- [x] **M5.2** — Per-item delete on pending list — _2026-04-27_
  - Scrollable SwiftUI panel listing pending waypoints / yaw updates / spheres / boxes / lassos with trash icon per row. Removal pushes onto undo stack. Hover-highlight 3D marker deferred (not blocking).
  - **Files:** `SplatViewerView.swift` (panel inline), `SplatImmersiveRenderer.swift` (`removePendingWaypoint(at:)` + per-type variants).

- [x] **M5.3** — Local draft / autosave — _2026-04-27_
  - `EditorDraftStore` writes JSON to `Application Support/CloudTour/drafts/<sceneId>.json` every 5 s. Resume-prompt on reopen, cancel-with-confirm, draft-discard on Save.
  - **Files:** new `cloudtour-avp/CloudTourVision/Viewer/EditorDraftStore.swift`, `SplatViewerView.swift` lifecycle hooks.

### P0 — Feature parity with web

- [x] **M5.5** — Set starting view — _2026-04-27_
  - View-mode button captures current head pose; renderer caches per-frame head + splat model and inverts to splat-local on snapshot. Persists via `scenes.default_camera_position` PATCH.
  - **Files:** `SplatImmersiveRenderer.swift` (`snapshotHeadPoseInSplatLocal()`), `SplatViewerView.swift` (button + Save wiring), `Models/Scene.swift` (new field), `Models/Waypoint.swift` (new `CameraPosition` struct).

### P1 — Calibrate ergonomics

- [x] **M5.6** — Numeric XYZ + yaw input panel — _2026-04-27_
  - Stepper panel for X/Y/Z (cm), uniform scale (%), yaw (°). Renderer.applyTransform pushes onto undo stack so numeric edits are reversible.
  - **Files:** `SplatViewerView.swift`, `SplatImmersiveRenderer.swift`.

### P1 — Waypoints

- [x] **M5.10** — Scene-link graph view — _2026-04-27_
  - List-style sections per source scene + orphan-scenes section, linked from `TourEditorView`. 2D force-directed layout deferred → M6.12.
  - **Files:** new `cloudtour-avp/CloudTourVision/Dashboard/WaypointGraphView.swift`, `Dashboard/TourEditorView.swift`.

### P1 — Display / debug toggles

- [x] **M5.11** — Display HUD — _2026-04-27_
  - Three toggles shipped (hide waypoints / hide pending deletions / hide silhouette) wired through `buildMarkers`. Opacity + point-size sliders deferred → M6.7 (needs Metal shader uniform).
  - **Files:** `SplatViewerView.swift` (HUD inline), `SplatImmersiveRenderer.swift` (`setDisplayFlags`, render-path gates).

### P2 — Onboarding / a11y / sim

- [x] **M5.20** — First-run onboarding overlay — _2026-04-27_
  - AppStorage-gated sheet with 5 sections (calibrate / waypoint / brush+box+lasso / undo / starting view). Shown once per user.
  - **Files:** `SplatViewerView.swift`.

- [~] **M5.21** — VoiceOver audit — _partial 2026-04-27_
  - A11y labels + hints added on critical buttons (sim pinch, dismiss-onboarding, etc.). Full sweep continued in M6.10.

- [~] **M5.22** — Simulator parity — _partial 2026-04-27_
  - Spacebar shortcut on sim pinch button. Box / lasso commit shortcuts continued in M6.11 (gesture-state simulation needs renderer extensions).

### P2 — SplatPainting evaluation

- [x] **M5.23-ADR** — SplatPainting evaluation ADR — _2026-04-27_
  - `.docs/architecture/splatpainting-evaluation.md` (ADR-001). Decision: do not integrate upstream (license non-commercial, CUDA stack, no API). Concept-borrow only via M6.17 (stamp tool).
