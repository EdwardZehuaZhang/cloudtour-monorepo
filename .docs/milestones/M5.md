# M5 — AVP Splat Editor: polish + parity

Closes the gaps surfaced in the M4.15 editor audit. Goal is "feels like a real editor" — undo, per-item control, hotspots, display toggles, autosave — without re-architecting the renderer.

## Priority bands

- **P0** — must ship to call M5 done. Blocks user trust in Save flow.
- **P1** — high leverage, ships in M5 if scope holds.
- **P2** — defer to M6 if pressed.

## Tasks

### P0 — Save-flow trust

- [x] **M5.1** — Undo / redo stack — _2026-04-27_
  - **Goal:** Cmd-Z (external keyboard) or two-finger-tap-back gesture undoes the last in-session edit (waypoint placement, brush sphere, box, lasso, calibrate-commit). Redo restores it. Stack flushes on Save.
  - **Files:** `cloudtour-avp/CloudTourVision/Viewer/SplatImmersiveRenderer.swift` (history struct + apply/revert), `SplatViewerView.swift` (UI + keybinding).
  - **Depends on:** none.

- [x] **M5.2** — Per-item delete on pending list — _2026-04-27_
  - **Goal:** SwiftUI panel listing pending waypoints / spheres / boxes / lassos with a trash icon per row. Highlights the corresponding 3D marker on hover.
  - **Files:** new `cloudtour-avp/CloudTourVision/Viewer/PendingEditsPanel.swift`, `SplatImmersiveRenderer.swift` (`removePending(id:)` + per-item id hash).
  - **Depends on:** M5.1 (shares history surface).

- [x] **M5.3** — Local draft / autosave — _2026-04-27_
  - **Goal:** Editor session persists pending edits to `URLSession`-local JSON every 5 s. On reopen offers "resume draft" before discarding. Cancel button warns if unsaved drafts exist.
  - **Files:** new `cloudtour-avp/CloudTourVision/Viewer/EditorDraftStore.swift`, `SplatViewerView.swift` (load/save lifecycle).
  - **Depends on:** M5.1.

### P0 — Feature parity with web

- [ ] **M5.4** — Hotspot tool
  - **Goal:** New `ToolMode.hotspot`. Aim + pinch places a hotspot; second pinch cycles content type (link / image / text). SwiftUI side-panel edits the chosen content. Save persists to `hotspots` table via existing BE route.
  - **Files:** `EditTool.swift` (case `.hotspot`), `SplatImmersiveRenderer.swift` (gesture + render), `SplatViewerView.swift` (picker + content panel), new `cloudtour-avp/CloudTourVision/Models/Hotspot.swift` if missing, BE wiring already exists at `cloudtour-be/src/routes/scenes/[sceneId]/hotspots`.
  - **Depends on:** M5.2.

- [x] **M5.5** — Set starting view — _2026-04-27_
  - **Goal:** "Set starting view from here" button while in `.view` mode captures current head pose → writes `tours.default_camera_position` on Save.
  - **Files:** `SplatViewerView.swift`, `SplatImmersiveRenderer.swift` (`snapshotHeadPose()`), `cloudtour-avp/CloudTourVision/ViewModels/TourDetailViewModel.swift` (PATCH).
  - **Depends on:** none.

### P1 — Calibrate ergonomics

- [x] **M5.6** — Numeric XYZ + yaw input panel — _2026-04-27_
  - **Goal:** Optional numeric stepper alongside gesture for fine calibrate (cm-precision translate, 1° yaw).
  - **Files:** `SplatViewerView.swift` (panel), `SplatImmersiveRenderer.swift` (`applyCalibrateDelta(_:)`).

- [ ] **M5.7** — Snap-to-floor + snap-to-grid
  - **Goal:** Toggle in calibrate panel. Floor snap aligns lowest splat-AABB face to y=0; grid snap quantises translation to 5 cm.
  - **Files:** `SplatImmersiveRenderer.swift` (snap logic in `applyTwoHandGesture` + `applyOneHandGesture`).

- [ ] **M5.8** — Axis gizmo overlay
  - **Goal:** While `.calibrate` is active, render a small XYZ gizmo at the splat origin so the user can see orientation drift without external cues.
  - **Files:** `SplatImmersiveRenderer.swift` (gizmo geometry alongside silhouette), shader registration.

### P1 — Waypoints

- [ ] **M5.9** — Arrival-yaw preview arrow
  - **Goal:** Before committing a yaw-update pinch, show a translucent arrow at the waypoint indicating the candidate yaw. Pinch confirms.
  - **Files:** `SplatImmersiveRenderer.swift`.

- [x] **M5.10** — Scene-link graph view — _2026-04-27_ (list-style; 2D layout deferred to M6)
  - **Goal:** Flat panel (not in-immersive) showing scenes as nodes, waypoints as directed edges. Click an edge to jump to the source scene + select that waypoint.
  - **Files:** new `cloudtour-avp/CloudTourVision/Dashboard/WaypointGraphView.swift`.

### P1 — Display / debug toggles

- [x] **M5.11** — Display HUD — _2026-04-27_ (toggles shipped; opacity / point-size sliders deferred — needs Metal shader uniform)
  - **Goal:** Collapsible panel with sliders/toggles for splat opacity, point-size multiplier, hide-waypoints, hide-pending-deletions, hide-silhouette. State persists per-session, not saved to BE.
  - **Files:** new `cloudtour-avp/CloudTourVision/Viewer/DisplayHUD.swift`, `SplatImmersiveRenderer.swift` (read flags in render path).

- [ ] **M5.12** — Performance counters
  - **Goal:** fps + draw-count overlay, toggled by HUD or shake gesture. Off by default.
  - **Files:** `SplatImmersiveRenderer.swift` (CADisplayLink frame timing), `DisplayHUD.swift`.

### P1 — Selection refinement

- [ ] **M5.13** — Erase-the-eraser
  - **Goal:** Modifier (long pinch) inside a pending deletion volume restores splats in that sub-region. Implemented as a positive-mask deletion that subtracts from the negative.
  - **Files:** `SplatImmersiveRenderer.swift` (DeletionMask diff), `EditTool.swift`.

- [ ] **M5.14** — Lasso projection-cone preview
  - **Goal:** While drawing the lasso, render a translucent extrusion of the polygon along the view direction so the user sees what will be erased before release.
  - **Files:** `SplatImmersiveRenderer.swift`.
