## Codebase Patterns
- pnpm workspaces with turbo for build orchestration; workspace packages use `workspace:*` protocol
- Root tsconfig.json has `strict: true` and `noUncheckedIndexedAccess`; each package extends it
- Tailwind CSS design tokens defined as CSS custom properties in `apps/web/src/app/globals.css` using oklch color space
- shadcn/ui components live in `packages/ui/src/`; `apps/web/src/components/ui/` re-exports from `@cloudtour/ui`
- `packages/ui` has its own `cn()` in `src/utils.ts`; apps/web keeps `@/lib/utils` for app-specific utilities
- Fonts: Cormorant Garamond (display via next/font/google) and Geist (UI via `geist` npm package)
- ESLint requires `@typescript-eslint/eslint-plugin` and `@typescript-eslint/parser` for no-explicit-any rule
- `NODE_ENV=production` is set in the shell; use `NODE_ENV=` prefix when running `pnpm install` or `pnpm add` to ensure devDependencies install
- Database types (`database.types.ts`) must include `Relationships: []` on every table for Supabase typed queries to work (without it, select results type as `never`)
- Dashboard routes use `(dashboard)` route group with `export const dynamic = "force-dynamic"` to prevent static generation (requires auth/env vars)
- Use Tailwind config token names (`duration-fast`) not arbitrary values (`duration-[var(--duration-fast)]`) to avoid ambiguous class warnings
- Swift models used as NavigationLink values need Hashable conformance — add it when the model will be used in navigation
- Pre-existing TS typecheck error in cloudtour-fe (getSafeThumbnailUrl) — does not block Swift-only changes
- Sidebar warm off-white text color: `oklch(85% 0.015 68)` — used as arbitrary Tailwind value `text-[oklch(85%_0.015_68)]`
- packages/types is types-only (no runtime code); packages/db and packages/ui export from `src/index.ts`
- `null` in PlanLimits means "unlimited"; plan limit constants belong in packages/db or apps/web, not packages/types
- All Supabase timestamp fields are typed as `string` (ISO 8601), not Date
- Migration SQL files live in `packages/db/migrations/` numbered sequentially (001_, 002_, etc.)
- RLS policies use `auth.uid()` and join through `org_members` for org-scoped access
- Published tours/scenes/waypoints/hotspots are publicly readable (no JWT); all other access requires org membership
- Slug deduplication uses sequential suffixes (-2, -3), never random — implemented in `generate_unique_slug()` DB function
- `billing_events` and `tour_views` inserts use service role (no user INSERT policy); reads are scoped to org members/owners
- Stripe client uses lazy init pattern in `apps/web/src/lib/stripe.ts` (same pattern as Resend in `email.ts`)
- Stripe webhook handler verifies `stripe-signature` before processing; idempotency via `billing_events.stripe_event_id` unique constraint
- Supabase clients: `createBrowserClient()` (client), `createServerClient()` (server w/ cookies), `createServiceClient()` (admin, server-only)
- Env validation uses Zod in `packages/db/src/env.ts`; public vars validated separately from server vars
- `@supabase/ssr` handles cookie-based auth for Next.js App Router server components
- API routes: use `createServerClient()` + `supabase.auth.getUser()` for JWT auth; return 401 if no user
- React 18 uses `useFormState` from `react-dom` (not `useActionState` from `react` which is React 19+)
- Use `useFormStatus` in a child `SubmitButton` component for pending state (must be inside `<form>`)
- Auth pages use `(auth)` route group with shared layout; auth callback is at `app/auth/callback/route.ts` (not in route group)
- Server actions with `redirect()` must not return a value after redirect — redirect throws internally
- Zod v4 `z.literal(true, "error message")` syntax (not `{ errorMap }` object)
- Org-scoped API routes use `requireOrgRole(orgId, minimumRole)` from `@/lib/api-utils` for auth + RBAC
- Plan limits constants live in `apps/web/src/lib/plan-limits.ts` as `PLAN_LIMITS` map
- Slug generation for tours uses `slugify()` from `@/lib/api-utils` + sequential dedup loop against DB
- Don't import `@supabase/supabase-js` directly in apps/web — use `Awaited<ReturnType<typeof createServerClient>>` for the client type
- Nested resource routes (scenes under tours) must verify parent ownership (tour.org_id === orgId) before any operation
- Storage bucket names and path conventions centralized in `apps/web/src/lib/storage.ts` — use STORAGE_BUCKETS and path helpers (splatFilePath, thumbnailPath, assetPath)
- @mkkellogg/gaussian-splats-3d has no @types — custom declarations live in `apps/web/src/types/gaussian-splats-3d.d.ts`
- SplatViewer component in `apps/web/src/components/viewer/` is shared between public viewer and editor contexts
- Dynamic import of heavy 3D libraries to keep initial bundle small; `sharedMemoryForWorkers: false` to avoid CORS requirements
- ViewerContext provides viewer instance + container ref to overlay children; overlay div uses `pointer-events-none` with markers as `pointer-events-auto`
- 3D→2D projection: `Vector3.project(camera)` → NDC → screen coords; reuse a single Vector3 to avoid GC in rAF loops
- Editor uses `(editor)` route group at `/editor/[tourId]` — separate from `(dashboard)` to avoid DashboardShell wrapping; middleware protects both `/dashboard/*` and `/editor/*`
- Editor components live in `apps/web/src/components/editor/`; TourEditor is the main orchestrator client component
- HTML5 native drag-and-drop for scene reorder (no external library); optimistic UI with background PATCH sync
- Waypoint/hotspot API routes nested under scenes: /scenes/[sceneId]/waypoints and /hotspots; PATCH/DELETE at /waypoints/[waypointId] and /hotspots/[hotspotId]
- Editor marker placement uses `useDebouncedCallback` (500ms) for position sync — CSS transform immediately, API updates debounced
- Screen-to-3D unprojection: NDC → unproject near/far → direction → place at fixed depth; reuse Vector3 instances to avoid GC in rAF loops
- EditorMarkersOverlay component manages 3D→2D projection + click-to-place + drag-to-reposition; lives inside SplatViewer as child (uses ViewerContext)
- Floor plan API: one floor plan per tour; GET uses `.maybeSingle()`, scene_positions are normalized 0-1 coords [{scene_id, x, y}]
- Private assets bucket images need long-lived signed URLs (e.g., 10 years) since there's no public CDN for the assets bucket
- Supabase jsonb columns return `Json` type — cast via `as unknown as TypedInterface` for typed usage in app code
- `Promise.resolve({ data: [] })` infers `never[]` — use explicit typed variables + conditional fetching instead
- `useSearchParams()` in Next.js 14 requires `<Suspense>` wrapper for pages that static-generate
- Public tour viewer at `/tours/[slug]` uses SSR `generateMetadata` for SEO; client component handles 3D viewer + interactions
- View count increment uses Supabase RPC (`increment_view_count`) for atomic updates; deduplication via hashed IP + tour_id
- Sheet (slide-over) component in `packages/ui/src/sheet.tsx` — uses same `@radix-ui/react-dialog` as Dialog, no new dependency
- Resend client must be lazily instantiated (not at module level) to avoid build failure when RESEND_API_KEY is missing
- Zod v4 `z.enum()` uses string message parameter, not `{ required_error }` object syntax
- Invite flow: org_members row created with `user_id: null`, `invited_email` set, `invite_token` auto-generated; on accept, `user_id` populated and `invite_token` nulled
- Blog content lives in `apps/web/content/blog/*.mdx`; utility in `apps/web/src/lib/blog.ts` reads/parses them with `gray-matter`
- MDX rendering uses `next-mdx-remote/rsc` for server-side rendering (zero client JS); `@tailwindcss/typography` for prose styles
- RSS feed at `/feed.xml` via route handler; blog posts statically generated via `generateStaticParams`
- AVP `EditorDraft` evolves with new optional fields — make new lists `Optional` with explicit `CodingKeys` so prior drafts on disk still decode (Swift's synthesised `Decodable` only tolerates missing keys for Optionals).
- AVP renderer mutation paths must call `Self.recordHistory(&state)` BEFORE mutating undoable surface (`pendingWaypoints`, `pendingHotspots`, deletions, transform). Skipping it leaves the undo stack incoherent.
- Swift 6 `@Sendable`-region captures: `OSAllocatedUnfairLock.withLock { $0 = ... }` cannot read `var` locals declared outside; hoist the computation into a `let` (use an IIFE-style closure if needed) and then assign that constant inside the lock.
- Non-Sendable Compositor types (`[LayerRenderer.Drawable]`, `[ReticleRenderer.Marker]`, `LayerRenderer.Drawable`): always extract `.count` (or other trivial scalars) into a local `let` before passing into `@Sendable` lock closures.
- SwiftUI `Canvas` `ctx.stroke(_:with:)` wants a `Color`, not a `TintShapeStyle`. Use `Color.accentColor.opacity(...)` (NOT `.tint.opacity(...)`).
- Files-picked URLs from `.fileImporter` are security-scoped: wrap reads in `startAccessingSecurityScopedResource()` / `stopAccessing…`, and copy bytes into the app's `temporaryDirectory` before deferring work to a later UI tick.

**Learnings for future iterations:**
  - `aabbLock.withLock { $0 = AABB(lo: lo, hi: hi) }` with `lo`/`hi` declared as `var` outside fails Swift 6 `@Sendable`-region capture. Pattern: hoist the computation into a `let` (e.g. an immediately-invoked closure that returns the value) and then assign that constant inside the lock. Same shape as the existing M5 `Dictionary(uniqueKeysWithValues:)` workaround.
  - `[LayerRenderer.Drawable]` is non-Sendable. Capturing `drawables.count` (or `markers.count`) inside `perfLock.withLock` requires extracting them into local `let` constants first; the lock closure is `@Sendable`.
  - SwiftUI `Canvas`'s `ctx.stroke(...with: .color(...))` only accepts a `Color`-typed shape style. `.tint` is a `TintShapeStyle` which the type-checker silently rejects with a confusing "static property 'tint' requires the types 'Color' and 'TintShapeStyle' be equivalent" message. Use `Color.accentColor.opacity(...)` instead.
  - For force-directed graph layouts at the scale of an AVP tour (≤ ~50 scenes), an O(N²) Fruchterman–Reingold loop with ~250 iterations runs synchronously on `onAppear` in <50 ms — no need to dispatch off-main, and no need for a real-time animation loop.
  - When the AVP needs file-side data (Files picker), the Files-picked URL is security-scoped: call `startAccessingSecurityScopedResource()` before reading, and copy the bytes into the app's `temporaryDirectory` so subsequent UI pauses can't strand you with a stale scope.
  - `Scene` is the name of both SwiftUI's protocol AND our domain model. In files that import SwiftUI but reference our `Scene` model in `ForEach`, the compiler may resolve the SwiftUI protocol — keep usages as `[Scene]` and pass through methods so type inference flows from the array element type.
