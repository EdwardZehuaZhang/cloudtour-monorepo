## Codebase Patterns
- pnpm workspaces with turbo for build orchestration; workspace packages use `workspace:*` protocol
- Root tsconfig.json has `strict: true` and `noUncheckedIndexedAccess`; each package extends it
- Tailwind CSS design tokens defined as CSS custom properties in `apps/web/src/app/globals.css` using oklch color space
- shadcn/ui initialized in `apps/web` with components at `src/components/ui/`; use `cn()` from `@/lib/utils` for class merging
- Fonts: Cormorant Garamond (display via next/font/google) and Geist (UI via `geist` npm package)
- ESLint requires `@typescript-eslint/eslint-plugin` and `@typescript-eslint/parser` for no-explicit-any rule
- `NODE_ENV=production` is set in the shell; use `NODE_ENV=` prefix when running `pnpm install` or `pnpm add` to ensure devDependencies install
- packages/types is types-only (no runtime code); packages/db and packages/ui export from `src/index.ts`
- `null` in PlanLimits means "unlimited"; plan limit constants belong in packages/db or apps/web, not packages/types
- All Supabase timestamp fields are typed as `string` (ISO 8601), not Date
